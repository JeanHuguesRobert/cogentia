# Trajectoire stratégique — Du snapshot GitHub au 27 septembre 2026

## 1. Position de départ

Le snapshot du 4 juillet 2026 montre que les deux derniers mois ont produit davantage qu'une accumulation de textes et de code.

Ils ont produit une architecture émergente :

```text
corpus
→ index
→ Guide
→ packets
→ continuations
→ agents sous mandat
→ traces
→ validation humaine
```

La priorité stratégique n'est donc plus de multiplier les idées. Elle est de convertir cette architecture en capacité réelle avant le 27 septembre 2026.

## 2. Deadline

Le 27 septembre 2026 fonctionne comme contrainte réelle.

Il ne faut pas traiter cette date seulement comme une échéance politique. Elle est aussi une épreuve de vérité pour l'Autonomie de Capacité :

> suis-je devenu plus capable d'agir, de publier, de répondre, de vérifier, de transmettre et de tenir une ligne cohérente grâce au corpus et au jumeau numérique ?

La campagne sénatoriale doit servir de banc d'essai, non de distraction.

## 3. Objectif opérationnel

Objectif : disposer au 27 septembre d'un jumeau numérique partiel mais réellement utile, capable de soutenir le travail politique, éditorial, documentaire et stratégique.

Le jumeau numérique doit pouvoir :

1. retrouver rapidement les sources du corpus ;
2. produire des synthèses fiables ;
3. préparer des réponses publiques ;
4. transformer les conversations en issues ou packets ;
5. maintenir une mémoire de campagne ;
6. signaler contradictions, oublis, risques et bifurcations ;
7. distinguer source, dérivé, cache et opérationnel ;
8. faciliter la publication multi-supports ;
9. soutenir la cohérence entre C.O.R.S.I.C.A., Institut Mariani, FractaVolta, Cogentia, Kudocracy, #1755, Corse2038 et les autres chantiers ;
10. laisser explicitement le dernier mot humain.

## 4. Principe directeur

Formule à conserver :

> Délégation maximale du travail ; non-délégation de la souveraineté.

Ou, sous forme plus opérationnelle :

```text
l'agent prépare ;
l'humain tranche ;
le corpus conserve ;
la trace vérifie ;
la correction améliore.
```

## 5. Axes de travail jusqu'au scrutin

### Axe A — Mémoire de campagne

Créer une mémoire structurée de campagne :

- positions publiques ;
- réponses à interlocuteurs ;
- textes Facebook / Substack / LinkedIn ;
- posts sur autonomie de papier / autonomie de capacité ;
- interactions avec Pierre Poggioli, élus, médias, institutions ;
- tableau des silences institutionnels ;
- sources et preuves.

Livrable recommandé :

```text
research/campaign/2026_senatoriales_memory.md
```

### Axe B — Guide public

Le Guide public doit devenir une porte d'entrée lisible.

Priorités :

- page courte : « Que peut faire le Guide ? » ;
- exemples de questions utiles ;
- avertissement sur les limites ;
- citations et sources visibles ;
- distinction entre corpus public et mémoire privée ;
- indication claire : l'IA propose, Jean Hugues Robert valide.

### Axe C — Packets d'interaction

Chaque interaction importante doit pouvoir devenir un packet :

- contexte ;
- interlocuteur ;
- enjeu ;
- source ;
- formulation proposée ;
- action réalisée ;
- suite ouverte.

Objectif : que les échanges publics ne disparaissent pas dans Facebook, Messenger, Gmail ou WhatsApp.

### Axe D — Workflow agentique

Stabiliser une règle simple :

| Cas | Règle |
|---|---|
| idée nouvelle | issue ou note courte |
| transformation substantielle | issue obligatoire |
| intervention d'agent de code | issue + branche/PR sauf exception explicite |
| texte public court | `main` direct acceptable avec source claire |
| dépôt sensible | aucun agent sans mandat explicite |

### Axe E — Snapshot hebdomadaire

Chaque semaine jusqu'au scrutin :

```text
Snapshot hebdomadaire
- faits nouveaux ;
- textes publiés ;
- décisions prises ;
- issues ouvertes / fermées ;
- contradictions détectées ;
- risques ;
- priorités de la semaine suivante.
```

## 6. Trois temporalités

### Court terme — 4 au 15 juillet

Objectif : stabiliser l'architecture narrative publique.

Actions :

- publier l'article sur le jumeau numérique ;
- créer la mémoire de campagne ;
- fixer les règles de workflow agentique ;
- relier le snapshot aux documents existants ;
- vérifier la lisibilité du Guide public.

### Moyen terme — 16 juillet au 31 août

Objectif : transformer le système en cadence productive.

Actions :

- produire un rythme de publications ;
- transformer chaque conversation importante en continuation ;
- enrichir les dossiers : autonomie, #1755, FractaVolta, transparence, anti-capture ;
- faire fonctionner les agents comme assistants de production, non comme auteurs souverains ;
- renforcer le retrieval et les citations.

### Dernière ligne — 1er au 27 septembre

Objectif : privilégier la clarté, la répétition, la preuve et la cohérence.

Actions :

- réduire les chantiers techniques non indispensables ;
- produire des messages publics stables ;
- consolider les preuves ;
- préparer réponses courtes et longues ;
- publier un bilan d'Autonomie de Capacité appliquée à la campagne.

## 7. Ce qu'il faut éviter

1. Construire une infrastructure qui ne publie pas.
2. Publier sans rattacher au corpus.
3. Laisser les agents produire sans mandat.
4. Multiplier les dépôts sans registre canonique.
5. Confondre efficacité technique et effet politique.
6. Traiter la sénatoriale comme une fin plutôt que comme un test.

## 8. Mesure de réussite

Le 27 septembre, la question ne sera pas seulement : « quel résultat électoral ? »

La question sera :

> Le système m'a-t-il rendu plus capable ?

Indicateurs :

- nombre de textes publics produits et reliés au corpus ;
- nombre de réponses préparées par le jumeau numérique puis validées humainement ;
- rapidité de retrieval des sources ;
- qualité des citations ;
- nombre d'interactions transformées en packets ;
- clarté des positions stabilisées ;
- réduction de la perte d'information ;
- capacité à reprendre un dossier après interruption ;
- capacité à expliquer publiquement la méthode.

## 9. Synthèse

La trajectoire stratégique est simple :

```text
Juillet : rendre le système lisible.
Août : rendre le système productif.
Septembre : rendre le système politiquement utile.
```

Le jumeau numérique n'est pas un luxe technique. C'est une réponse pratique à une contrainte : un humain seul ne peut pas tout tenir sans mémoire, sans trace, sans méthode et sans délégation contrôlée.

## 10. Formule de clôture

> L'Autonomie de Capacité commence ici : devenir capable de faire plus, mieux, plus vite, avec davantage de mémoire, de preuve et de contrôle, sans abandonner la responsabilité humaine.
