# Mémoire de campagne — Sénatoriales 2026

## 1. Objet

Ce document sert de mémoire structurée pour la campagne sénatoriale 2026.

Il ne remplace pas les documents sources, publications, messages, issues, courriers, posts ou échanges. Il les relie, les résume, les qualifie et les rend reprenables.

Fonction principale : transformer le flux de campagne en corpus exploitable.

```text
interaction
→ contexte
→ position
→ source
→ réponse
→ trace
→ suite ouverte
→ validation humaine
```

Règle centrale :

> L'agent prépare ; Jean Hugues Robert tranche ; le corpus conserve ; la trace vérifie ; la correction améliore.

## 2. Deadline

Date de référence : **27 septembre 2026**.

La campagne sénatoriale sert de banc d'essai de l'Autonomie de Capacité appliquée au système de pensée, de publication, de retrieval, d'action et de traçabilité.

Question de clôture :

> Le système a-t-il rendu le candidat plus capable d'agir, de répondre, de publier, de vérifier, de relier et de transmettre ?

## 3. Principes de campagne à maintenir

### 3.1. Autonomie de Capacité

Formulation courte :

> Une autonomie de papier donne des mots. Une autonomie de capacité donne des moyens réels d'agir, de contrôler, de produire, de transmettre et de corriger.

Applications :

- institutions contrôlables ;
- énergie locale ;
- capacité numérique ;
- transparence des actes publics ;
- autonomie communale et territoriale ;
- formation des habitants ;
- souveraineté pratique, non symbolique seulement.

### 3.2. Pas de pouvoir sans contrôle

Formulation courte :

> Plus on donne de pouvoir, plus il faut de contrôle, de trace et de révocabilité.

Applications :

- élus ;
- mandataires ;
- prestataires ;
- agents numériques ;
- institutions territoriales ;
- projets énergétiques ;
- budgets publics.

### 3.3. Transparence comme infrastructure

Formulation courte :

> La transparence ne doit pas dépendre du courage individuel. Elle doit devenir une infrastructure impersonnelle, versionnée et reproductible.

Applications :

- analyse des actes ;
- tableaux de bord publics ;
- suivi des silences institutionnels ;
- contrôle des promesses ;
- mise en visibilité des asymétries d'information.

### 3.4. Délégation sans abdication

Formulation courte :

> Délégation maximale du travail ; non-délégation de la souveraineté.

Applications :

- jumeau numérique ;
- agents IA ;
- rédaction assistée ;
- recherche de sources ;
- gestion de campagne ;
- génération de packets ;
- validation finale humaine.

### 3.5. Possibilisme

Formulation courte :

> Il ne s'agit pas de promettre l'idéal. Il s'agit d'explorer rationnellement les possibles, de tester les chemins, d'écarter les impasses et de continuer.

Applications :

- propositions réalistes mais non résignées ;
- expérimentation territoriale ;
- droit à l'essai ;
- boucles action → réaction → correction ;
- refus du fatalisme corse.

## 4. Messages publics stabilisés

### 4.1. Autonomie de papier / Autonomie de Capacité

Version courte :

> J'oppose une autonomie de papier à une autonomie de capacité. La première est formelle. Elle ne garantit rien. La seconde commence lorsque nous devenons réellement plus capables qu'avant.

À utiliser pour :

- posts courts ;
- réponses Facebook ;
- vidéos ;
- interventions sur l'autonomie institutionnelle.

### 4.2. Sénatoriale comme banc d'essai

Version courte :

> Cette campagne n'est pas seulement une candidature. C'est un test : peut-on construire, avec peu de moyens mais beaucoup de méthode, une capacité politique plus transparente, plus documentée, plus contrôlable ?

À utiliser pour :

- blog ;
- Substack ;
- présentation de méthode ;
- lien avec Cogentia / jumeau numérique.

### 4.3. Le Petit Parti / Pertitellu

Version courte :

> Le Petit Parti n'a pas vocation à capturer le pouvoir. Il sert à rendre visibles des idées, des méthodes et des contrôles que les grandes machines politiques évitent trop souvent.

À vérifier / compléter avec documents source du dépôt concerné.

### 4.4. Corse capable

Version courte :

> Une Corse capable n'est pas une Corse qui reçoit un statut et attend. C'est une Corse qui apprend à produire, vérifier, décider, transmettre et corriger.

À utiliser pour :

- FractaVolta ;
- Corse2038 ;
- formation ;
- énergie ;
- démocratie directe ;
- transmission patrimoniale.

## 5. Registre des interactions publiques

Chaque interaction significative doit être résumée selon le format suivant.

### Template d'interaction

```markdown
### YYYY-MM-DD — Interlocuteur / canal / sujet

- **Canal** : Facebook / Substack / email / réunion / téléphone / autre
- **Interlocuteur** : nom public ou catégorie
- **Sujet** : sujet principal
- **Contexte** : pourquoi cette interaction compte
- **Position défendue** : formulation courte
- **Sources liées** : liens internes / externes / documents du corpus
- **Réponse produite** : résumé ou lien
- **Statut** : publié / brouillon / répondu / à reprendre / clos
- **Suite ouverte** : prochaine action
- **Sensibilité** : public / privé / à anonymiser
```

### Interactions à intégrer en priorité

- Pierre Poggioli — autonomie, fractures du mouvement national, stratégie institutionnelle.
- Mona Redmoor — statut d'autonomie, Constitution, autonomie réelle ou communication politique.
- Cuntestà Podcast — autonomie, référendum, capacité.
- Parlementaires corses — courriers et relances.
- Institutions liées au projet #1755 — demandes, silences, réponses, suites.
- Interactions autour de FractaVolta — énergie, Guide public, seconde vie solaire, commune pilote.
- Interactions autour du Guide public — questions des visiteurs, limites, sources, corrections.

## 7. Registre des positions

### 7.1. Autonomie institutionnelle

Position : soutenir toute avancée réelle, mais refuser de confondre reconnaissance formelle et capacité effective.

À documenter :

- texte constitutionnel ;
- question référendaire ;
- pouvoirs transférés ;
- contrôles associés ;
- moyens financiers ;
- transparence ;
- capacité citoyenne à vérifier.

### 7.2. Référendum

Position : la formulation de la question référendaire sera un indicateur de sincérité politique.

À documenter :

- formulation exacte lorsqu'elle sera connue ;
- effets juridiques ;
- ambiguïtés ;
- possibilités de manipulation ;
- pédagogie publique.

### 7.3. Énergie et FractaVolta

Position : l'énergie est un champ prioritaire d'Autonomie de Capacité.

À documenter :

- potentiel solaire ;
- stockage ;
- Fractanet ;
- Guide public ;
- seconde vie solaire ;
- commune pilote ;
- réseaux de batteries ;
- contrôle citoyen et territorial.

### 7.4. Démocratie directe / liquide

Position : la délégation politique doit devenir explicite, traçable, révocable et contrôlable.

À documenter :

- Kudocracy ;
- mandats ;
- vote vivant ;
- un vivant = une voix dans les périmètres pertinents ;
- contrôle des représentants ;
- limites et objections.

### 7.5. Transparence anti-capture

Position : les systèmes capturés ne se réforment pas par morale seule ; ils doivent être rendus vérifiables.

À documenter :

- actes publics ;
- conflits d'intérêts ;
- asymétries d'information ;
- tableaux de bord ;
- protocoles d'observation ;
- précautions contre l'accusation infondée.

## 8. Registre des packets de campagne

Un packet de campagne est une unité de reprise.

Il doit contenir :

```yaml
packet_type: campaign_interaction.v1
date: YYYY-MM-DD
subject: ""
actor: ""
channel: ""
context: ""
claim_or_question: ""
corpus_sources: []
response_draft: ""
validated_response: ""
status: open|published|answered|closed|deferred
next_action: ""
human_final_word: true|false
sensitivity: public|private|anonymize
```

Objectif : éviter que les échanges de campagne restent prisonniers des plateformes.

## 10. Distinction public / privé

Ce document est conçu comme mémoire publique ou semi-publique de campagne.

Règles :

- ne pas inclure de données personnelles sensibles ;
- anonymiser les personnes privées ;
- citer les sources publiques ;
- renvoyer les éléments sensibles vers un registre privé si nécessaire ;
- ne pas publier de brouillons stratégiques sensibles sans validation humaine.
